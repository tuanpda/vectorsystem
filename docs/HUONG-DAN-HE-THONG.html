1. Hệ thống là gì?

Biến PDF/DOCX thành tri thức tìm kiếm được (vector) cho chatbot.

Thành phầnVai trò
Admin UI (:5173)Upload, Parse, Index, Test RAG, API Keys
Knowledge API (:3000)API trung tâm, queue, RAG
MinerU (:8000)Parse PDF → Markdown + JSON
DockerPostgres pgvector :5433, Redis :6380, MinIO :9000
OpenAIEmbedding text-embedding-3-small (Index + RAG)
App BotPOST /rag/query + API Key mk_live_...

2. Luồng dữ liệu

[Admin UI] ──JWT──► [API :3000] ──► MinIO / Postgres / Redis │ └──► [MinerU :8000] khi Parse [Bot] ──API Key──► POST /rag/query ──► contexts ──► Bot gọi LLM trả lời

3. Ba bước xử lý tài liệu

BướcNútCầnKết quả
UploadUploadAPI + DockerFile → MinIO
ParseParse+ MinerUMD + JSON → MinIO
IndexIndex+ OpenAIVector → Postgres

Trạng thái Sẵn sàng = RAG tìm được.

4. Khởi động — một lệnh (khuyến nghị)

cd D:\CODE\MINERU\mineru-knowledge-admin
.\start-all.ps1

Mở Docker + 3 cửa sổ: MinerU, API, Admin. Dừng: .\stop-all.ps1

Điều kiện: Docker Desktop đang chạy; đã có api\.env (OPENAI_API_KEY, ADMIN_*, …).

7. API Key cho chatbot

  1. Admin → API Keys → Tạo key → copy mk_live_... (một lần).
  2. Bot gọi:
POST http://127.0.0.1:3000/api/v1/rag/query
X-API-Key: mk_live_...
{"question":"...","documentIds":["uuid"],"topK":8}

Key chỉ dùng RAG; không upload/parse/index.
